# 配布用 2 ステッププロンプト

LP 仕様（`lp-spec.md`）が固まっている人に、そのまま渡せる汎用プロンプト。
このキット（Claude Code）が使える環境なら `lp-builder` スキルが自動で走る。使えない環境でも、下の手順だけで進められる。

---

## Prompt 1 / 縦長画像スライスを作る

```text
以下のLP構成を、スマホLP用の縦長画像にしてください。
もし lp-builder スキルが使える環境なら、そのスキルの Phase 1 を使ってください。
使えない場合も、以下のワークフローに沿って進めてください。

目的:
すでに決まっているLP構成を、高品質なスマホLPビジュアルにしたいです。
画像生成は GPT Image 2.0（gpt-image-2 / OpenAI API・`OPENAI_API_KEY` 必要）を使ってください。
まず画像でLPの見た目を作り、あとでHTMLでCTAボタン・動画枠・フォーム・リンクを機能化する前提です。

参考デザイン:
添付したアイキャッチ／キービジュアル／過去LP／デザインスクショをトンマナの参考にしてください。
配色・余白・見出しの強さ・フレーム・人物/商品画像の扱い・全体の雰囲気を優先して引き継いでください
（レイアウトの丸写しではなくトンマナの踏襲）。

デザイン方針:
- スマホ表示を最優先にする
- 縦に接続できる細長い画像を4〜8枚程度に分けて作る
- 全画像で同じデザインシステムを維持する
- 1枚のポスターではなく、実際に読めるLPとして設計する
- 情報がしっかり入ってくる読みやすさを優先する
- 見た目のインパクトは出すが、派手にしすぎない
- CTA・動画・フォーム・リンクになりそうな場所は、後からHTMLボタンを重ねやすい位置に置く
- ページ番号・過剰装飾・小さすぎる文字・強すぎるネオン・情報の詰め込みは避ける
- 日本語テキストの精度が重要なセクションは、文字を画像に焼き込まず、後でHTMLで載せる前提の余白にする

LP構成:
---
ここに lp-spec.md のセクション構成を貼ってください。
（目的＝CTA1つ／ターゲット1人／オファー／各セクションの見出し・本文・CTA文言・トンマナ）
---

出力:
- セクションごとに分かれた縦長画像を生成してください
- 生成した全画像の保存パスを教えてください
- つなげたときに1本のスマホLPに見えるようにしてください
```

---

## Prompt 2 / HTML で LP 化する

```text
HTML化工程に進んでください。
もし lp-builder スキルが使える環境なら、その Phase 2 を使ってください。
直前に生成したLP画像スライスを使って、スマホファーストの静的LPをHTML/CSS/JSで作ってください。

重要:
- 画像は再生成しないでください
- 生成済み画像をLPのセクションスライスとして使ってください
- 見た目は画像、機能はHTMLで担当する形にしてください

要件:
- index.html を作る
- 生成画像はプロジェクト内の assets/ フォルダに保存する
- 画像スライスを縦に隙間なく接続する
- LP幅は最大430pxにする（width: min(100%, 430px)）
- スマホでは画面幅いっぱいに表示する
- PCでは430pxのスマホキャンバスとして中央に表示する
- 画像内のCTA部分に透明なHTMLボタンを重ねる
- 動画・フォーム・リンクになりそうな部分にも透明なHTMLボタンを重ねる
- リンク先URLが未確定の場合は、きれいな仮モーダルを開く
- リンク先URLが確定している場合は、正しいURLを設定する
- コンバージョンが目的なら、画面下に固定CTAを置く
- 画像内テキストは見た目用なので、重要な内容は sr-only のHTMLテキストとしても入れる
- 画像のデザインを邪魔しない、静かなHTMLスタイルにする

動作:
- CTAは実リンク、または仮モーダルを開く
- 動画枠は実動画、または仮モーダルを開く
- モーダルは閉じるボタン・背景クリック・Escキーで閉じられるようにする

検証（QA）:
- 390pxと430pxのスマホ幅で確認する
- 横スクロールが出ないか確認する
- PCでスマホLPが中央表示されるか確認する
- すべてのボタンとモーダルが動くか確認する
- スマホプレビューとPCプレビューのスクショを保存する

出力:
- 完成したローカルファイルパス、または公開URL
- スクショの保存先
- 検証結果の簡単な報告
```

## Prompt 3 / デプロイ（公開）

```text
完成したスマホLPを公開（デプロイ）してください。
- デプロイ先は Vercel（CLI / Vercel MCP を接続済みなら「create a deployment」でプレビュー公開）か、GitHub Pages。
- 公開は取り消せない外部公開なので、実行前に必ず私に確認してください。
- 公開後、本番URLが200を返すこと、計測タグ（GA4等）が発火することを確認して、URLを教えてください。
```

## メモ

- 画像生成は **GPT Image 2.0（OpenAI API・APIキー要）** が既定。大量生成は OpenAI API、別モデルは Runway / HiggsField も可（`reference/connections.md`）。
- この 3 プロンプトは、キット無しでも「画像で見た目・HTML で機能・デプロイ」の流れを再現するためのもの。
- キットを使う場合は、壁打ち（`lp-grilling`）→ `/goal` ループ（`prompts/goal-loop.md`）で
  ヒアリングと QA・デプロイまで含めて自走させるほうが、抜けが少なく仕上がる。
