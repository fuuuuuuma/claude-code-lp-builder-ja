これは何のプレゼント？

Claude Codeに渡すと、いきなり作らず「壁打ち」から始まる

  • 受け取った人がClaude Codeにこのキットを渡すと、AIがまず質問攻め（壁打ち）を始める 事業内容・LPの内容・参考URL…を一問ずつ。答えるだけでLPの設計図が固まる
  • 固まった設計図をもとに、スマホファーストのLPを自動で組み上げ、デプロイまで GPT Image 2.0（OpenAI API・キー要）で画像→HTMLで機能化→QA→Vercel/GitHubへ公開
  • 「LP作りたいけど何からか分からない」を、渡すだけで前に進める形にした

02

なぜ「壁打ち → 自走」の順番なのか

いきなり作らせると
  • 前提がズレたまま完成してしまう
  • コピーがふわっとして刺さらない
  • 手戻りが多く、何度も作り直し
→
先に壁打ちすると
  • 「誰の・何を」が鮮明になる
  • 確定コピーから作るので芯がブレない
  • QAは機械的に合否が出る＝自走できる

※ コピーの良し悪し（主観）は人間が壁打ちで決め、ビルドとQA（客観）はループに任せる ＝ maker ≠ checker

壁打ちで聞くこと（質問例）

04

中身② /lp-goalループ lp-goal

固めた仕様を「契約」にして、3体以上のサブエージェントが各観点100点を出すまで自走させる仕組み。採点はサブエージェント（maker≠checker）。結果と検証を固定するのがループ工学のコツ。

  • Done＝何ができたら完了か
  • Evidence＝何で確かめるか（QA）
  • Constraints＝何を壊してはいけないか
  • Stop＝どこで止まるか（上限/成功）

※ goal-setter / makeloop（MIT）のループエンジニアリングを土台に、LP制作向けに用意

中身④A 同梱ノウハウ：LP作成のコツ

  • ファーストビューで「誰の・何が・どうなる」を一目で（冒頭で離脱させない）
  • 1画面1アクション。CTAは1つに絞り、要所で繰り返す
  • PASONA等の型に沿って、悩み→共感→解決→証拠→行動の順で並べる
  • 証拠（実績・お客様の声・データ）で信頼を作る。誇張・断定保証はしない
詳細は reference/lp-playbook.md（出典つき） ノウハウA15
06

中身④B 同梱ノウハウ：AIでLP作る注意

  • GPT Image 2.0は文字描画が高精度（日本語OK・公称約99%）。焼き込みも実用的に
  • ただし画像の限界（後で編集しづらい／SEO弱／重い）は残る→重要テキストはHTML
  • 計測タグ・フォーム/LINE連携・プライバシー表記を忘れない
  • 生成画像の権利・商標に注意。公開＝外部送信は人間が確認
詳細は reference/ai-lp-notes.md（出典つき） ノウハウB16

使い方（Claude Codeでの始め方）

  • 1Claude Codeでこのフォルダ（プロジェクト）を開く（CLAUDE.md自動読込・画像はOpenAI API gpt-image-2）
  • 2「LP作って」と話しかける → プランモードで壁打ちが始まる
  • 3事業内容・LPの内容・参考URL…の質問に答えていく
  • 4/lp-goal → 3体以上のサブエージェントが各観点100点を出すまで自走
  • 5「GitHub か Vercel か」を聞かれたら選ぶ → その先へ公開（デプロイ）まで

07

安全とルール

キットに組み込んだ歯止め

  • 取り消せない操作（公開・デプロイ・課金・決済/フォーム連携・外部送信）の前は必ず人間に確認
  • コピー・数値は事実から。誇張・断定保証をしない（景表法／薬機法／特商法／ステマ規制）
  • 仕様（lp-spec.md）にない機能・コピーを足さない（スコープ厳守）
  • QAせず「完成」と言わない（ゲートが通ってから完了報告）
