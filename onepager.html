中身① 壁打ち（lp-grilling）

  • 質問は一度に1つ。混乱させない
  • 各質問に推奨案を添える（Yes/修正で返せる）
  • 決定木を1枝ずつ降りて、抜けを潰す
  • 参考URLがあれば、聞く前にまず読む
  • 結論を lp-spec.md に書き出す
GrillMe 由来

Matt Pocock の「容赦ない一問一答で計画を研ぐ」手法（MIT）を、LPの仕様詰め向けに再構成。原典リンクはフッター。

壁打ちで聞くこと（質問例）
  • 事業内容→何を売る／誰の役に立つ事業？
  • LPの内容→目的・オファー・ゴール（CTAは1つ）
  • ターゲット→誰の・どんな悩み・どんな状態？（1人に絞る）
  • 参考URL→トンマナ（色・フォント・憧れるLP）
  • 証拠・制約→実績／お客様の声、期日・法規制

全カテゴリは .claude/skills/lp-grilling/references/question-bank.md に。

中身② /lp-goalループ（自走）

固めた仕様を「契約」にして、3体以上のサブエージェントが各観点100点を出すまで自走させる仕組み。採点はサブエージェント（maker≠checker）。結果と検証を固定するのがループ工学のコツ。

D

Done

何ができたら完了か

E

Evidence

何で確かめるか（QA）

S

Stop

どこで止まるか（上限/成功）

送る契約のイメージ
# 壁打ちで lp-spec.md が固まったら、壁打ちが固まったら /lp-goal を実行
/lp-goal lp-spec.md の仕様どおりのスマホLPを、qa-checklist.md の全項目に合格する状態まで作り切る。Done＝index.html生成・390/430pxで横スクロール無・全CTA/モーダル/フォームが動作・スクショ目視済み。仕様にない機能やコピーは足さない。公開前は人間に確認。

※ goal-setter / makeloop（MIT）のループエンジニアリングを土台に、LP制作向けに用意。

中身④ 同梱ノウハウ

LP作成のコツ
  • FVに4要素（コピー/画像/CTA/信頼）
  • 1ページ＝1ゴール＝1主要CTA
  • 新PASONA等の型で並べる
  • フォーム項目は8以下、可能なら5以下
  • 誇張・断定保証をしない（景表法/薬機法）
AIでLP作る注意
  • GPT Image 2.0は文字描画が高精度（日本語OK・約99%）
  • ただし画像の限界（編集不可/SEO/重い）は残る→文字はHTML
  • CTAは透明ボタンでなく<a>/<button>
  • 計測タグ・フォーム/LINE連携を確認
  • 生成画像の権利・商標に注意

※ 数値は出典つき。自分のLPは必ずA/Bで再検証。詳細は reference/lp-playbook.md ／ reference/ai-lp-notes.md。

7

使い方と安全

  • 1Claude Codeでこのフォルダ（プロジェクト）を開く（CLAUDE.md自動読込・画像はOpenAI API gpt-image-2）
  • 2「LP作って」と話しかける → プランモードで壁打ちが始まる
  • 3事業内容・LPの内容・参考URL…の質問に答える
  • 4/lp-goal → 3体以上のサブエージェントが各観点100点を出すまで自走
  • 5「GitHub か Vercel か」を聞かれたら選ぶ → その先へ公開（デプロイ）まで
安全（キットに組み込んだ歯止め）

取り消せない操作は人間が確認

  • 公開・デプロイ・課金・決済/フォーム連携・外部送信の前は必ず確認（自動公開しない）
  • コピー・数値は事実から。誇張・断定保証をしない（景表法／薬機法／特商法／ステマ規制）
  • 仕様（lp-spec.md）にない機能・コピーを足さない（スコープ厳守）
  • QAせず「完成」と言わない（ゲートが通ってから完了報告）
